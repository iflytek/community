## What Are Discussions For?

Discussions are the place for open-ended conversations, community building, and questions that don't fit into issue tracking. Use Discussions for:

* Asking questions about the project
* Brainstorming new ideas
* Sharing showcase projects
* General community conversation

## Categories

| Category | Purpose |
|---|---|
| **General** | Open conversation about the community |
| **Ideas** | Brainstorm and discuss new features |
| **Q&A** | Ask for help or support |
| **Show and Tell** | Share projects built with the community's work |
| **Announcements** | Official project updates |

## How to Start a Discussion

1. Choose the right category
2. Write a clear, descriptive title
3. Provide context and details in the body
4. Be open to feedback and conversation

## Discussion Etiquette

* Be respectful and welcoming
* Stay on topic
* Use reactions (👍 👎 ❤️ etc.) to show agreement without cluttering threads
* If a discussion leads to a concrete action item, open an issue and link it
* If your question is answered, mark it as answered (for Q&A category)

## When to Use Issues vs Discussions

| Use Discussions When... | Use Issues When... |
|---|---|
| You have a question or need help | You found a bug |
| You want to brainstorm an idea | You have a specific feature request to implement |
| You want to share something | You want to track concrete work |
| You're not sure where to start | You have a clear action plan |
